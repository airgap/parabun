---
title: @para/pipeline
description: Runtime helpers for the |> pipeline operator. Detects affine chains over typed arrays and collapses them where beneficial.
---

The `|>` operator is a [Para language extension](/parabun/language#pipelines) parsed by Parabun's transpiler. At runtime, fused pipelines route through `@para/pipeline` which can analyse affine chains and compose them.

```ts
import simd from "@para/simd";

// Affine chain
const result = data
  |> simd.mulScalar(_, 2)
  |> simd.addScalar(_, 5)
  |> simd.sum;
```

Each `|>` step is one pass over the typed array. For very long chains where two adjacent affine ops are wasteful, `@para/pipeline.compile()` can fuse them.

## API

```ts
import pipeline from "@para/pipeline";

const fused = pipeline.compile([
  ["mulScalar", 2],
  ["addScalar", 5],
]);
const out = fused(data);   // single pass equivalent to: addScalar(mulScalar(data, 2), 5)
```

For most workloads you won't need the explicit `compile` — the runtime handles individual `simd.*` calls efficiently and the JS engine inlines the obvious cases.
